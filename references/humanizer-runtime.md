# 去 AI 味运行模块

> 每篇公众号正文写完后必读。目标不是把文章改花，而是降低“低质 AIGC / 机械稿 / 报告腔”风险，让读者觉得是一个真人在讲清楚一件事。

## 总原则

- 只改“怎么说”，不改事实和结论。
- 先删空话，再打散句式，最后补真人节奏。
- 不追求文学化。公众号要清楚、自然、有一点判断。
- AI 味风险为高时，不进入排版发布。

## 一眼像 AI 的高危句式

出现就优先处理：

- “不是 A，而是 B”
- “不仅是 A，更是 B”
- “这意味着什么”
- “值得注意的是”
- “需要指出的是”
- “综上所述 / 总而言之”
- “首先 / 其次 / 最后”
- “一方面 / 另一方面”
- “随着……的发展”
- “在……的背景下”
- “从某种意义上说”
- “不可否认 / 毋庸置疑”
- “让我们来看看”

处理方式：能删就删，能直接说结论就直接说结论。不要把否定铺垫当深度。

## 高频 AI 词和空话

高频出现时替换或删除：

- 此外、同时、然而、因此
- 本质上、说白了、换句话说
- 赋能、重塑、升级、打造、助力
- 深刻改变、重要意义、影响深远
- 前所未有、关键一步、全新格局
- 行业人士认为、专家表示、媒体报道称但不给来源

替换原则：

- 抽象词换成具体事实。
- 宏大词换成具体影响。
- 模糊主体换成具体公司、产品、平台、岗位、用户。

## 标点策略

不绝对禁用标点，但控制机械密度：

- 冒号可以用，但不要每个小节都写成“概念：解释”。
- 破折号少用；如果只是 AI 式补充说明，改成逗号或拆句。
- 双引号尽量改成「」；不要满篇引号。
- 连续列表不要太长。能用自然段说清楚，就不用机械 bullet。

## 真人感改写方法

### 1. 把空泛判断落到具体处

差：
> 这将对行业产生深远影响。

好：
> 真正受影响的是那些原本靠人工审核、人工排班、人工客服撑起来的小团队。

### 2. 开头少铺垫，先给读者一个抓手

差：
> 随着人工智能技术的快速发展，各行各业都迎来了新的变化。

好：
> 今天最值得看的不是某个模型又刷榜了，而是一批公司开始把 AI 从演示视频塞进真实工作流。

### 3. 用一点第一人称，但不要过量

可用：

- 我看到这里的第一反应是……
- 我更在意的是……
- 这个地方有点反直觉。
- 这事儿麻烦就麻烦在……

不要每段都“我觉得”。一篇 1500-3000 字，2-4 处足够。

### 4. 长短句混合

关键判断可以单独成段。

比如：

> 这就是风险。
>
> 不是技术做不到，而是很多公司还没想清楚，出了问题到底谁负责。

### 5. 承认复杂性

真人不会永远斩钉截铁。可以写：

- 这件事不能简单说好或坏。
- 它确实提高效率，但也把一些成本转移给了普通人。
- 我不确定它最后会走到哪一步，但现在这个变化已经足够明显。

## 三遍处理流程

### 第一遍：删空话

删除或替换：

- 意义重大、影响深远、值得期待
- 赋能、重塑、打造新格局
- 没有来源的专家/业内人士说法
- 重复解释同一个观点的段落

### 第二遍：打散结构

检查：

- 是否有“首先/其次/最后”
- 是否每段长度差不多
- 是否连续小标题都是抽象词
- 是否每节都用冒号开头
- 是否出现“不是 A，而是 B”超过 1 次

处理：

- 删连接词，让段落靠内容自然相连。
- 小标题改成具体问题或具体判断。
- 把太整齐的三段式拆成自然推进。

### 第三遍：补真人节奏

补 2-4 处即可：

- 一个第一人称判断
- 一个短句停顿
- 一个具体场景
- 一个承认复杂性的句子

不要为了“像人”强行口嗨。`xiaocong` 可以克制一点，`yeluzi` 可以更生活化一点。

## 终稿评分

发布前写给自己看，不要放入正文：

```text
AI 味风险：低 / 中 / 高
空话密度：低 / 中 / 高
口语自然度：低 / 中 / 高
是否需要重写：是 / 否
```

判定标准：

- **低**：读起来像真人，有具体信息，连接词不机械，可以发布。
- **中**：有几处报告腔或空话，局部再改。
- **高**：像摘要、像报告、像模板稿，必须重写。

## 最后 8 问

- 第一段有没有具体事件或场景？
- 有没有 2 个以上来源或事实支撑？
- 有没有“此外 / 值得注意的是 / 综上所述”残留？
- 有没有一堆“赋能 / 重塑 / 深刻改变”？
- 有没有模板式小标题？
- 有没有连续长段，读起来喘不过气？
- 有没有至少一个只有这篇文章才有的具体信息？
- 我读完像是在听人说话，还是在看机器总结？
