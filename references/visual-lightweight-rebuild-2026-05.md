# 公众号视觉轻量化重构说明（2026-05）

> 用途：沉淀本轮关于“保留现有审美，但显著降低制作重量”的重构共识。

## 1. 用户反馈的核心不是审美问题，而是制作成本问题

用户明确表示：
- 喜欢现有封面图和正文图的风格
- 但不接受每次生成都像在“做一个小项目”

因此后续优化方向不是换风格，而是：
- 保留视觉系统
- 降低变量数量
- 降低每次制作的决策成本
- 降低模板对所有场景通吃的执念

---

## 2. 封面图：轻变量、强模板

封面图适合继续保持单模板主路径。

### 推荐保留变量
- 主标题
- 副标题 / kicker
- 账号人格标签
- impact 数字
- 少量标签 pill

### 不建议继续增加的内容
- 大段正文信息
- 过多英文眉注
- 多层解释说明
- 与正文图重复的结构表达

### 结论
封面问题本质上不是模板不足，而是不要让封面承担太多正文信息。

---

## 3. 正文图：不要再追求“一个模板通吃”

当前正文图的根本问题：
- 变量太多
- 骨架太硬
- 一个模板想覆盖所有文章类型
- 每次像在填一张复杂配置表

### 新建议：三层结构

#### 第一层：统一视觉骨架
固定不变：
- 背景
- 字体
- 配色
- 卡片样式
- 标题层级
- 留白与边距

#### 第二层：模块化内容单元
建议长期保留的模块：
- `hero`：总量级冲击图
- `people`：人物 / 公司 / 角色卡
- `structure`：结构图 / 权力地图
- `compare`：表面 vs 本质 / 名单 vs 结构 / 访问 vs 牌局
- `quote`：少量观点强化
- `metric`：少量数字补充

说明：
- `quote` / `metric` 应是轻量补充，不应再是默认强制主图
- 不是每篇都要把所有模块做一遍

#### 第三层：文章编排器
根据文章类型决定只用 1~2 个模块：
- 日常分析稿：`hero + structure`
- 人物盘点稿：`people + structure`
- 数据冲击稿：`hero + compare`
- 观点评论稿：`compare + quote`

目标：
- 不再默认 3 张重图起步
- 让图跟着内容类型走，而不是让内容去迁就模板

---

## 4. 变量最小化原则

### hero
只保留：
- 一个大数字 / 核心判断
- 一个标题
- 一句解释

### people
只保留：
- 名字
- 角色
- 一句代表性描述

### structure
只保留：
- 3~4 个结构块
- 一句总结

### compare
只保留：
- 左右两列的对照标题
- 各自 1~2 句短说明

原则：
- 图解决表达问题，不承载整段正文
- 能短就短，能标签化就不写整段

---

## 5. 推荐的长期演化方向

1. 保留现有商业编辑风，不换系统
2. 封面继续单模板主路径
3. 正文图从“固定重模板”改为“模块库拼装”
4. 默认每篇 1~2 张图，不强迫上满
5. 以后优先减少变量，而不是增加变量
