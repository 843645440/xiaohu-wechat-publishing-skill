# 公众号"低创作度 + 高风险限流"治理改造设计

> 日期：2026-06-21
> 范围：`xiaohu-wechat-publishing` skill + 早/晚两个 Hermes Agent 定时任务提示词
> 目标：消除微信"低创作度"标签、规避政治财经类高风险限流，恢复推流

---

## 0. 名词说明（供外部 Agent 审阅，不假设读者熟悉本仓库）

- **Skill**：`xiaohu-wechat-publishing`，一套公众号写作/排版/配图/发布的规则 + 脚本集合，`SKILL.md` 是它的运行契约。
- **Cron 提示词**：两个 Hermes Agent 定时任务（早 07:00 全球热点、晚 18:00 国内热点），每天自动抓新闻、调用上面这个 Skill 写稿并推草稿箱。
- **双账号**：`xiaocong`（熵增时刻，产业/科技/信息密度）与 `yeluzi`（思想的野路子丶，资本/冲突/情绪）。
- **publish_pipe.py**：发布主入口，发布前做硬校验（已有 AI 身份披露黑名单扫描）。
- 本文档区分两层规则：**Skill 通用规则**（所有公众号任务都适用）与 **Cron 任务追加规则**（只在定时任务里生效）。

## 1. 问题诊断

微信"低创作度内容"四条规则，与当前生产逻辑逐条对应：

| 平台规则 | 当前命中点 |
|---|---|
| (1) 同质化、缺信息增量 | 双账号每天围绕**同一热点**各写一篇 → 两篇高度相似；早晚两次 + 固定模板 → 模式可识别 |
| (2) 搬运/洗稿/大篇幅引用 | 核心流程是"抓单条新闻 → 重构改写"，易判洗稿 |
| (3) 信息量不足 | `writing-persona.md` 全程震惊体/信息差/情绪推进 → 被当营销号低信息量 |
| (4) 低价值 AIGC | **每天定时、批量、双账号**自动生成 → 批量 AIGC 特征最强信号 |

额外风险：早间定位（美股/黄金/原油/加密/政策/地缘）与晚间（宏观/A股/基金/楼市/监管）**整体是政治财经高风险题材**，叠加限流。

**根因结论**：限流主因不是"财经题材"本身，而是 **定时批量 AIGC + 双号同题 + 单源洗稿 + 震惊体低信息密度** 四者叠加，再加高风险角度。

---

## 2. 已确认的改造方向（用户决策）

1. **风险规避力度 = B 软规避**：财经/科技题材保留，只从"对普通人影响"切入；不评判政策、不预测大盘、不碰敏感时政地缘。
2. **去同质化 = 双账号换不同选题**（事件/主题层面不同，非同题不同角度）。
3. **降频 = 双号各 1 篇/天（共 2 篇）**，从每天最多 4 篇降到 2 篇。
4. **写作风格 = 转向信息增量**：砍标题党与空震惊，改为独家信息整合 + 真实分析 + 原创观点。
5. **错峰（本设计拍板）**：早 07:00 只产 `xiaocong` 1 篇、晚 18:00 只产 `yeluzi` 1 篇，每号自选题材、天然错峰。
6. **代码校验（本设计拍板）**：`publish_pipe.py` 加高风险关键词 **warning 级软扫描**，不阻断发布。

## 3. Skill 层改造

### 3.1 新增 `prompts/quality-and-risk.md`（最高优先级，覆盖旧人格冲突部分）

定位：当 `writing-persona.md` / `ai-tech-writing-guide.md` 与本文件冲突时，**以本文件为准**。`SKILL.md` 的"先读什么"里写明：写正文前必读本文件。

#### A. 信息增量硬标准（治"低创作度"）

- **多源交叉**：每篇 **≥3 个独立信息源**，禁止单篇新闻改写；单一来源贡献 >70% 内容 = 判洗稿，重写。
- **原创分析层**：除事实陈述外，必须含以下至少 **2 种**——横向对比、历史纵向对比、具体数据测算、对普通人的可操作结论。
- **信息密度**：用具体数字、公司、产品、案例、场景支撑，禁止用情绪形容词堆砌充数。
- **终稿自检（必答）**："这篇给了读者哪一条别处看不到的信息或判断？"答不出 → 不发。

#### B. 去标题党（治"营销号判定"）

- 禁止：纯震惊体空转（"真相和你想的不一样"但无实质反转）、无证据颠覆、夸张承诺式标题。
- 标题改为 **信息型 / 价值型**：准确反映正文，可带观点，不夸大。
- 开头禁止"近日…发布"流水账，但也禁止纯制造危机感的空喊；用一条具体事实/数据切入。

#### C. 风险软规避硬规则（治"限流"）

绝对不做：

1. 评判 / 预测 / 解读**具体政策走向**（财政、货币、监管、楼市等政策本身的对错与方向）。
2. 预测**股市 / 汇市 / 币市涨跌**，喊点位，给买卖建议。
3. 碰**敏感时政、地缘冲突站队、领导人、群体性事件、社会对立**议题。
4. 传未经证实消息（防谣言文）；所有数据须有来源。

软规避改写方式：财经/科技题材**只从"对普通人工作/消费/生活的影响"切入**。

高风险 → 安全切口示范：

| 高风险写法 | 软规避改写 |
|---|---|
| 央行降息释放什么信号、利率走向预测 | 利率变了，普通人的存款和房贷该怎么算 |
| 美联储加息与中美博弈 | 美元利率变化，海淘/留学/换汇成本怎么变 |
| A股后市怎么走、该不该抄底 | 行情波动时，普通人常见的几个认知误区 |
| 某政策出台意味着监管转向 | 这项变化落到具体一个家庭/个体身上是什么 |

**高风险关键词清单**（选题与终稿都要比对，命中需改切口或换题）：
降息/加息/货币政策/财政政策/监管收紧/政策转向/抄底/牛市/熊市/点位/涨停/暴跌/地缘/制裁站队/领导人姓名/群体事件 等（实现时落成可维护列表）。

#### D. 去同质化（治"同质化"）

- 双账号**同日选题在事件/主题层面必须不同**，不是同题不同角度。
- 发布前比对 `publish-history`：核心实体 + 文章结构高度相似 → 拒发。

### 3.2 改写 `prompts/writing-persona.md`

- 删除/降级"信息差+震惊体（泛人群杀手锏）"整节与"标题套路"中的标题党模板。
- 保留：节奏、对话感、数据有体感、结构清晰这些**不违规的可读性技巧**。
- 顶部加一行：本文件服从 `quality-and-risk.md`，冲突以后者为准。

### 3.3 调整 `references/ai-tech-writing-guide.md`

- 降"震惊体/制造危机感/爆文"调性，保留"行业理解、产业链拆解、对普通人影响"这些真正产生信息增量的要求。
- "观点重构式创作"明确补一句：**必须基于多源**，单源重构视为洗稿。

### 3.4 更新 `SKILL.md`

- "先读什么"：写正文前必读 `prompts/quality-and-risk.md`。
- "写作快路径"：加入信息增量自检、单源洗稿自检、风险软规避三条。
- "配图快路径"无需大改。
- "发布前硬校验"：补充高风险关键词软扫描项（见 3.5）。

### 3.5 （拍板纳入）`publish_pipe.py` 高风险软扫描

- 与现有 `check_ai_disclosure()` 并列，新增高风险关键词扫描函数。
- dry-run 与正式发布时命中 → 打印 **warning**（默认不阻断），列出命中词与所在段落，提醒人工复核。
- 预留开关：可配置为 block（默认 warn）。
- 关键词来源：`quality-and-risk.md` 中 C 节清单落成的可维护列表（建议放配置或单独常量）。

## 4. Cron 提示词改造

### 4.1 结构性变化

- **早 07:00**：只为 `xiaocong`（熵增时刻）产 1 篇，偏科技/产业/AI 等较安全题材。
- **晚 18:00**：只为 `yeluzi`（野路子）产 1 篇，与早间事件/主题不同；财经题材走软规避切口。
- 每号每天 1 篇、共 2 篇、错峰。
- 选题阶段增加**风险筛 + 信息增量评分**；候选排序从"传播潜力"改为"价值增量 + 传播"综合分。
- 去重：72h 去重 + 双账号当日互斥。
- 报告增项：信息源数量、信息增量点、风险规避自检结果。

### 4.2 早间任务改写后提示词（完整，可直接替换）


## 6. 验证方式

- 用一条历史选题手工跑早/晚提示词，检查：是否多源、是否触发风险淘汰、终稿自检是否答得出信息增量点。

- 现有 `tests/` 单测保持通过；如为软扫描加最小单测（命中/未命中各一例）。
