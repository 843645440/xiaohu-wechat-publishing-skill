# 创作质量与风险规避（最高优先级规则）

> 适用：所有公众号写作任务（双账号通用）。
> **优先级**：本文件与 `writing-persona.md`、`references/ai-tech-writing-guide.md` 冲突时，**一律以本文件为准**。
> 目的：避开微信"低创作度内容"标签与政治/财经类高风险限流。

---

## 为什么有这份规则

微信对"低创作度内容"不推荐，包括：同质化、搬运/洗稿/大篇幅引用、信息量不足、低价值 AIGC（借新技术批量生成低信息量内容）。
政治、宏观财经、政策、地缘等高风险角度极易被限流。
旧的"震惊体 / 信息差 / 标题党 / 情绪推进"打法正好命中这些判定，必须改为**信息增量 + 软规避**。

---

## A. 信息增量硬标准（治"低创作度"）

每篇文章发布前必须同时满足：

1. **多源交叉**：至少引用并整合 **≥3 个独立信息源**。严禁基于单篇新闻改写。
   - 单一来源贡献内容 **>70%** → 判定洗稿，重写。
   - "整合"指交叉印证、补背景、做对比，不是把多篇拼在一起。
2. **原创分析层**：除了陈述事实，必须包含以下至少 **2 种**：
   - 横向对比（同类公司/产品/事件的比较）
   - 历史纵向对比（这件事在更长时间线上的位置）
   - 具体数据测算（用数字算出一个别人没直接给的结论）
   - 对普通人的可操作结论（读者看完能做什么/注意什么）
3. **信息密度**：用**具体数字、公司、产品、案例、场景**支撑论点；禁止用情绪形容词堆砌充字数。
4. **终稿自检（必答一句）**：
   > "这篇给了读者哪一条别处看不到的信息或判断？"
   答不出来 → **不发，重写**。

---

## B. 去标题党（治"营销号判定"）

- **禁止**：
  - 纯震惊体空转（标题喊"真相和你想的不一样"，正文没有实质反转）
  - 无证据的颠覆式断言
  - 夸张承诺、贩卖焦虑式标题
- **标题改为信息型 / 价值型**：准确反映正文内容，可带观点，但不夸大、不做不兑现的承诺。
- **开头**：禁止"近日，××发布了……"流水账；也禁止纯制造危机感的空喊。用**一条具体事实或数据**切入。

> 可读性技巧（节奏、对话感、数据有体感、结构清晰）仍然保留，见 `writing-persona.md`——但服务于"把信息讲清楚"，不是制造情绪噱头。

---

## C. 风险软规避硬规则（治"限流"）

### 绝对不做（命中即换切口或换题）

1. 评判 / 预测 / 解读**具体政策走向**（财政、货币、监管、楼市等政策本身的对错与方向）。
2. 预测**股市 / 汇市 / 币市涨跌**，喊点位，给买卖建议。
3. 碰**敏感时政、地缘冲突站队、领导人、群体性事件、社会对立**议题。
4. 传未经证实的消息（防谣言文）；所有关键数据必须可溯源。

### 软规避方式

财经 / 科技题材**只从"对普通人工作 / 消费 / 生活的影响"切入**。题材可以留，高风险角度去掉。

| 高风险写法（不要） | 软规避改写（这样写） |
|---|---|
| 央行降息释放什么信号 / 利率走向预测 | 利率变了，普通人的存款和房贷该怎么算 |
| 美联储加息与中美博弈 | 美元利率变化，海淘 / 留学 / 换汇成本怎么变 |
| A 股后市怎么走 / 该不该抄底 | 行情波动时，普通人常见的几个认知误区 |
| 某政策出台意味着监管转向 | 这项变化落到一个具体家庭 / 个体身上是什么 |

### 高风险关键词（选题与终稿都要比对）

命中高风险词，必须改切口或换题。**关键词清单见 `data/high-risk-keywords.json`**（单一真相源）——调整词表只改这个文件，无需改代码或本文档。当前清单覆盖：降息/加息/货币政策/财政政策/监管收紧、抄底/牛市/熊市/涨停/暴跌/点位、地缘/制裁/站队/脱钩/领导人/群体事件 等。

> `publish_pipe.py` 会在发布前做高风险关键词**软扫描**（warning 级，不阻断），命中会列出提醒人工复核（含"大盘"等易误报词的提示）。看到 warning 不要无视——确认是否需要改切口。

## D. 去同质化（治"同质化 / 框架一致"）

微信低创作度里的"同质化"不只看文字是否重复，也看**同一个账号近期内容**的整体重复感：主题、标题句式、正文框架、封面构图、正文图风格是否短期内高度相似。

- **同账号近期去重优先**：写作前比对 `~/.hermes/workspaces/wechat/publish-history.md` 与 `publish-history.jsonl`，同一账号近 7 天内不得连续复用同一套标题句式、正文结构、封面原型、正文图类型。
- **双账号同日选题也要不同**：事件 / 主题层面必须不同，不是同一事件换角度。
- **信息组件固定，段落顺序不固定**：每篇都要有事实、多源、原创分析、读者价值、风险边界，但不能每天按同一顺序填空。
- **通用结构小标题禁用**：不要把"事实底座 / 背景解释 / 原创分析 / 普通人影响 / 中国变量 / 克制结论 / 这件事意味着什么"当固定小标题。小标题必须是内容专属句。
- **封面与正文图也算同质化信号**：同账号近 7 天不得反复使用同一封面版式、同一大字排布、同一画面主体、同一 `baoyu-comic + dramatic/neutral` 风格组合。
- 发布前比对：
  - 核心实体（公司 / 人物 / 事件）+ 标题句式 + 文章结构与近 7 天同账号已发高度相似 → **拒发，换题或换结构**。
  - 正文有信息增量但封面只是同模板换字 → **重做封面**。
  - 结构安全但正文图连续同风格、同构图、同隐喻 → **减少正文图或换图型**。

## E. 去AI味检查（治"AIGC检测 / 机械感"）

> 来源：humanizer / unclecheng-reduce-ai-perception-v2，整合自 stop-slop、Humanizer-zh、khazix-writer 等。

微信和第三方AIGC检测工具会识别AI写作痕迹。即使内容质量过关，机械感过强仍可能被标记或让读者感到"这不是人写的"。

### 核心原则

- **改最少，效果最大**：能改一个词就不改一句，能删一句就不重写一段。
- **保留创作意图**：只改"怎么说"，不改"说什么"。
- **注入灵魂**：目标不是无菌文字，而是「有见识的普通人在认真聊一件打动他的事」。

### 写正文时必须遵守

1. **禁用标点**：正文中不使用冒号`：`、破折号`——`、双引号`""`。引用用`「」`或直接不加引号。
2. **禁用句式**：不用"不是A，而是B"（直接写B）、"，带着……"万能状语、"仿佛/犹如/宛若"。
3. **禁用连接词**：不用"此外""同时""值得注意的是""需要指出的是""综上所述""首先/其次/最后""一方面/另一方面""不可否认""毋庸置疑""随着...的发展""在...的背景下"。
4. **禁用填充短语**：删掉"值得一提的是""从某种意义上说""事实上""不得不说""坦率地说""众所周知""毫无疑问"。
5. **高频踩雷词**：不用"说白了""意味着什么""本质上""换句话说""不可否认"。
6. **句式断裂**：在情绪高点或转折点，用极短句独立成段制造停顿。不能每段都这样，但全文至少2-3处。
7. **口语化表达**：使用"怎么说呢""你品品""你说离不离谱""我寻思了一下""太离谱了""你敢信"等口语词组（详见 `prompts/banned-words.md` 第十节推荐词组）。
8. **第一人称**：适当用"我"，"我看到这儿的时候愣了一下""我觉得更有意思"说明有个真人在想。
9. **承认复杂性**：真人有矛盾心理，"这东西确实厉害，但总让人有点不舒服"比"这令人印象深刻"强。
10. **允许一些乱**：跑题、旁白、半成型的想法才是人。完美结构像算法。

### 处理流程

写完初稿后，按以下流程检查：

**Phase 1：扫描**
对文本快速扫描，标记问题位置。按优先级排序：
1. 最毒句式（"不是A而是B"、"带着..."、"心中涌起..."）
2. 禁用词（加载 `prompts/banned-words.md`）
3. 标点违规（冒号、破折号、双引号）
4. 结构问题（加载 `prompts/structures.md`）
5. 节奏/风格问题

**Phase 2：诊断分级**

| 指标 | 轻度 | 中度 | 重度 |
|------|------|------|------|
| 禁用词密度（/千字） | ≤5 | 6-15 | >15 |
| 连续排比段数 | ≤2 | 3-4 | ≥5 |
| 平均段落句数 | ≤3 | 3-5 | >5 |

取最高档位。任一达重度即按重度处理。

**Phase 3：执行**
- **轻度** → 仅 Pass 1（去泛化）
- **中度** → Pass 1 + Pass 2（去书面化）
- **重度** → 完整三遍 + 重点段落重写

**Pass 1：去泛化**
- 禁用词替换为具体动作/细节
- 心理描写外化："他很紧张" → "他的手在抖"
- 连续排比打断（保留1-2个）
- 标点修复：冒号→逗号，破折号→删或逗号，双引号→「」或不加

**Pass 2：去书面化**
- 句式去套路（加载 `prompts/structures.md`）
- "不是A，而是B" → 直接写 B
- 连接词精简（"此外""然而""因此" → 删或换隐性连接）
- 修饰词清扫（一次只用一个形容词，不连用不堆砌）

**Pass 3：回自然感**
- 节奏打碎：长短句交错，段落参差，一句话自成一段制造重点
- 对话去腔调：加入口语化表达
- 结尾去升华：用动作/场景收尾，不用总结/感慨
- 补具体感官细节
- 允许一些不完美：跑题、题外话、半成型想法
- 论述中的故意打破：用口语打断破坏严谨性，让论述有温度

### 个性与灵魂注入

**有观点。** 不要只报告事实——对其做出反应。"我真不知道该怎么看这件事"比中立罗列优缺点更像人话。

**变化节奏。** 短促有力的句子。然后是长一点的、不急不忙的。混着来。

**承认复杂性。** 真人有矛盾心理。"这东西确实厉害，但总让人有点不舒服"比"这令人印象深刻"强。

**适当时用"我"。** 第一人称不是不专业——是诚实。"我一直在琢磨..."或"有个事让我挺在意的..."说明有个真人在想。

**允许一些乱。** 完美结构像算法。跑题、旁白、半成型的想法才是人。

**对感受要具体。** 不是"这令人担忧"，而是"凌晨三点那些 agent 还在跑，没人盯着，想想就有点发毛。"

**论述中的故意打破。** 在逻辑论述中故意加入口语打断，破坏严谨性，让论述有温度。如重复强调（"就是单纯的，就是想看看"）、中途模糊（"我就不说是谁了"）、省略和打断（"我寻思了一下我没寻思明白"）。

**句式断裂。** 用极短的句子或短语独立成段，制造停顿和重量感。不能每段都这样，但在情绪高点或转折点用一下，效果极强。

**层层剥开。** 不直接讲结论，用"现象→表面解释→更深追问→核心洞察"渐进展开，让读者跟着你一起思考。

### 四层自检体系

**L1 硬性规则检查**
- L1-1 禁用词扫描：全文搜索禁用词，出现则必须替换
- L1-2 禁用标点扫描：全文搜索冒号、破折号、双引号，出现则必须替换
- L1-3 结构性套话扫描：检查"让我们来看看..."、"在当今...的时代"等套话
- 通过标准：三项扫描零命中

**L2 风格一致性检查**
- L2-1 开头检查：是否从具体、当下事件切入？
- L2-2 节奏与结构检查：是否有长短句交替？是否有一句话独立成段的"断裂"效果？
- L2-3 口语化检查：是否使用了推荐口语化词组？有没有至少一处自嘲或承认不足？
- L2-4 标点禁令二次确认：全文是否完全没有冒号、破折号和双引号？
- 通过标准：L2-1全部通过，L2-2至少3/4通过，L2-3至少3/4通过，L2-4通过

**L3 内容质量检查**
- L3-1 观点支撑检查：每个核心观点是否有具体人/场景/细节/数据支撑？
- L3-2 知识输出方式检查：知识点是否以"聊着聊着顺手掏出来"的方式呈现？
- L3-3 对立面与同理心检查：讲核心观点时，是否有对对方立场的理解和承认？
- 通过标准：L3-1和L3-2必须全部通过，L3-3至少通过相关项

**L4 活人感终审**
以读者视角通读全文，回答核心问题："读完这篇文字，我感觉是一个真人在跟我聊，还是一个AI在输出信息？"
- L4-1 温度感：情绪表达是体感记忆还是知识性描述？
- L4-2 独特性：是否有"只有这个作者才会写出来的角度"？
- L4-3 姿态检查：语气是不是"一个有见识的普通人在认真聊"？
- L4-4 心流检查：从头到尾读，有没有注意力断掉、需要回头理解逻辑的地方？
- 通过标准：整体感觉"这像是真人写的"

### 完整禁用词和句式清单

- `prompts/banned-words.md`：中文AI味禁用词与句式表（含标点禁令、高频踩雷词、推荐口语化词组）
- `prompts/structures.md`：结构性问题清单（二元对比、否定列举、公式化结构等）
- `prompts/examples.md`：改写示例库（含网文、技术、营销、学术场景）

### 自检

写完正文后快速扫描：
- 全文有没有冒号、破折号、双引号？→ 全部替换
- 有没有"不是A，而是B"？→ 直接写B
- 有没有"此外""然而""因此"超过2处？→ 删或换隐性连接
- 有没有连续3个以上相同长度的句子？→ 打断一个
- 有没有"首先...其次...最后"？→ 打散，用隐性连接
- 读起来像真人在聊天，还是AI在输出信息？→ 觉得AI味太重就返工

## 一页速查（写前贴在脑子里）

- 写之前：题材安全吗？切口是"对普通人的影响"吗？命中高风险词了吗？
- 找资料：凑齐 ≥3 个独立来源了吗？
- 写的时候：有没有 ≥2 种原创分析？有没有具体数字 / 案例？
- 写标题：是信息型还是标题党？
- 发之前：答得出"别处看不到的那一条"吗？同账号近 7 天结构 / 标题 / 封面 / 图型撞了吗？
