# 轻量标题与封面文案规则

> 适用：每篇文章定稿后、生成封面前。目标是标题短、主体清楚、封面不拥挤。

## 产物

在 job 目录写 `title-card.json`：

```json
{
  "article_title": "楼下超市，成了巨头的新战场",
  "cover_title": "楼下超市",
  "cover_subtitle": "外卖烧钱之后",
  "highlight": "新战场"
}
```

`article.md` 的 H1 使用 `article_title`。封面只使用 `cover_title`、`cover_subtitle` 和 `highlight`，不要直接塞完整长标题；不要再为封面生成三段分类小字。

## 文章标题

- 建议 12-22 个汉字，最多 26 个汉字。
- 标题只抓一个核心冲突或变化，不写成一句完整摘要。
- 不用“震惊、炸裂、赢麻了、全网沸腾”等低质词。
- 不用固定句式批量套娃。

## 强主体必须点名

如果正文核心围绕一个强识别主体，标题要点名，不要只写抽象品类。

- 单一公司/平台/产品占正文主要篇幅：标题必须出现主体名。
- 多主体对比：标题最多点名 1-2 个最关键、最有识别度的主体。
- 行业综述：不要硬塞公司名。
- 正文只是顺带提到：不要把公司名写进标题。

常见强主体：英伟达、苹果、华为、小米、阿里、腾讯、字节、OpenAI、Anthropic、智谱、DeepSeek、特斯拉、比亚迪、美团、京东、抖音、小红书。

示例：

- 弱：`芯片巨头，开始改写 AI 服务器价格`
- 强：`英伟达，正在改写 AI 服务器价格`
- 弱：`国产大模型，又走到关键路口`
- 强：`智谱，为什么被推到台前`

## 封面文案

- 封面按“视觉宽度”控制，不按纯汉字数硬砍。1 个汉字约等于 1，英文/数字约等于 0.5，空格约等于 0.3。
- `cover_title`：主视觉词。放最吸睛的主体：公司名、产品名、版本号、地点名。建议视觉宽度 ≤ 7，最多 ≤ 9。
- `cover_subtitle`：背景动作。放“谁做了什么 / 什么东西上线”。建议视觉宽度 ≤ 12，最多 ≤ 15；可以保留英文产品名，但不要写完整句子。
- `highlight`：点击钩子。放价格、冲突、结论或变化。建议视觉宽度 ≤ 6，最多 ≤ 8。
- 超长时优先压缩 `highlight`，其次压缩 `cover_subtitle`；不要压缩掉 `cover_title` 里的核心主体名。

示例：

```json
{
  "article_title": "CSDN上线GLM5.2 Coding Plan，海外版贵两倍多",
  "cover_title": "GLM5.2",
  "cover_subtitle": "CSDN上线Coding Plan",
  "highlight": "49元起"
}
```

长标题要压缩成“主标题 + 副标题”：

- 原：`外卖烧了几千亿之后，巨头们盯上了你家楼下的超市`
- 文章标题：`楼下超市，成了巨头的新战场`
- 封面主标题：`楼下超市`
- 封面副标题：`外卖烧钱之后`
- 高亮词：`新战场`

## 发布前自检

- 标题里是否有读者会主动关心的主体？
- 如果是公司新闻，公司名是否出现在标题或封面主标题里？
- 封面主标题是否一眼能读完？
- 封面是否没有把完整文章标题硬塞进去？
