# Markdown 元素速查

> 写正文时用这页即可。目标是信息清楚，不要为了排版加载长示例库。

## 可用元素

| 场景 | 优先用法 |
|---|---|
| 主章节 | `## 小标题` |
| 章节内小点 | `### 小标题` |
| 关键判断 | `> 这一句单独拎出来` |
| 对比、参数、清单 | GFM 表格 |
| 步骤、并列要点 | 有序/无序列表 |
| 产品名、命令、参数 | 行内代码 `` `xxx` `` |
| 配置、命令片段 | fenced code block |
| 大段切换 | `---` |
| 提醒/避坑 | `> [!tip] 标题` 后接引用内容 |

## 写稿要求

- 一篇文章至少出现 3 种非纯段落元素，常用组合：引用 + 列表 + 表格。
- 能用表格说清楚的 A/B 对比，不生成正文图。
- 不要堆连续长段落；连续 3 段以上都是纯段落时，改成列表、引用或表格。
- 不为了炫技使用代码块；只有命令、配置、伪代码真的有用时才写。
- 图片不要直接重复正文信息；正文图由视觉阶段按需生成。

## 最小示例

```markdown
两个变化最值得看：

| 维度 | 过去 | 现在 |
|---|---|---|
| 成本 | 单独买工具 | 打包订阅 |
| 用户 | 重度开发者 | 普通创作者也能试 |

> 真正的变化不是多了一个套餐，而是 AI 编程工具开始按渠道重新分层。

- 对个人：先看价格和使用频率。
- 对团队：先看权限、模型和结算方式。
```
