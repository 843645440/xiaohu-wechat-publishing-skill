# Markdown 元素手册（写正文时必须知道）

> **写文章时 agent 要主动用这些元素，而不是默认写成"段落 + 段落 + 段落"。**
> `scripts/format.py` 把下面所有元素都做了精心样式化，全部默认可用。

排版引擎：`markdown` 扩展 `tables` + `fenced_code` + `nl2br`，再叠加主题色与 inline style。
所有元素在所有主题里都已样式化（边框、斑马纹、color accent、行高），**不会出现"渲染丑"导致只能写段落**这种情况。

---

## 可用元素一览（agent 写稿时优先考虑）

| 元素 | Markdown 语法 | 渲染效果 | 何时该用 |
|---|---|---|---|
| 一级标题 | `## XXX` | 大字 + 主题色描边 / 渐变 | 主章节切分 |
| 二级标题 | `### XXX` | 小一号、左竖线 | 章节内的小段 |
| 段落 | 直接写 | 1.75 行高 + 字间距 | 默认载体 |
| 加粗 | `**xxx**` | 主题色 + 加粗 | 关键词 / 反转点 |
| 斜体 | `*xxx*` | 偏灰斜体 | 引语 / 命名 |
| 行内代码 | `` `xxx` `` | 浅灰底单行 | 命令、变量名、产品 ID |
| 代码块 | ```` ```lang \n ... \n ``` ```` | 深色背景 + 等宽字体 | 命令片段、配置、示意伪码 |
| 引用 | `> xxx` | 左侧主题色竖线 + 浅底色 | 引述、关键判断、读者视角的反问 |
| 提示容器 | `> [!tip] 标题` 后续 `> 内容` | 高亮卡片 | 边栏知识、关键提醒、读者操作建议 |
| 无序列表 | `- xxx` | 主题色圆点 | 平行枚举 |
| 有序列表 | `1. xxx` | 主题色编号 | 步骤、排序 |
| 表格 | 标准 GFM 表格 | 渐变表头 + 斑马纹 + 边框 | 对比、规格、参数、清单 |
| 分隔线 | `---` | 主题色短横 | 大段落切换 |
| 链接 | `[文字](url)` | 主题色下划 | 外部引用 |
| 图片 | `![alt](path)` | 圆角 + 阴影 | 见 visual-design.md |

## 必用建议（针对当前两个账号）

写一篇 2000–4000 字的稿子时，**应当出现至少 3 种非段落元素**：

1. **对比类内容用表格，别画图** —— "A vs B" 四个维度以内用 GFM 表格最干净。表格能讲清的，就不要画对比图。
2. **关键判断用引用（blockquote）** —— 不要把"这背后的逻辑是 xxx"写成普通段落，套个 `> ` 让读者一眼抓到。一篇文章 2–4 处即可。
3. **步骤、流程、清单用有序/无序列表** —— 别写成"首先……其次……再然后……"流水段落，列表比段落短一半还更清晰。
4. **命令、产品名、英文专词用行内代码** —— `gpt-5`、`tool_calls`、`/usr/bin/python3.12`、`MCP server` 这些都该是行内代码而不是裸文字。
5. **真正涉及配置 / 调用的，用代码块** —— 公众号读者其实很爱看具体配置和命令片段，比抽象描述可信。
6. **大段落切换用 `---`** —— 不要 3 个 `###` 标题挤在一起，节奏会断。

## 反例 vs 正例

### 反例：纯段落堆叠（80% 的 agent 默认产出）

```
Hermes 的押注非常直接。它把所有筹码放在 Agent 框架的工程化能力上，做模块系统、做工具发现、做权限模型。OpenClaw 选了相反的路，它押在生态扩展性上，做插件市场、做社区贡献入口、做轻量内核。这两条路并不互斥，但谁先撑到下一轮融资还很难说。
```

### 正例：信息分层

```markdown
两个项目走了完全相反的路：

| 维度 | Hermes | OpenClaw |
|---|---|---|
| 押注 | 工程化能力 | 生态扩展性 |
| 内核 | 重 | 轻 |
| 抽象层 | 模块 + 工具发现 + 权限模型 | 插件市场 + 社区入口 |
| 赌赢条件 | Agent 框架被企业标准化 | 插件生态自然涌现长尾 |

> 这两条路并不互斥。但融资窗口只有一次，谁先撑到下一轮，谁就有定义权。
```

读者从纯段落里挑出 4 维度对比要 30 秒，从表格里 5 秒就抓住了。这就是排版的价值。

## Agent 在写稿时该怎么用

写稿流程里，写每一段前先停一下问自己：

- 这段话是不是在做"对比" → 直接写表格
- 这段话是不是在列"步骤" → 直接写有序列表
- 这段话有没有一句"核心判断" → 把那一句拎出来套 `> `
- 段落里有没有英文专词 / 命令 / 产品名 → 全部包成 `` ` ` ``
- 章节内连着 3 段以上都是纯散文 → 强制至少塞 1 个非段落元素

写完整篇文章后通读：**如果整篇都是 `<p>` 段落，没有一个表格、列表、引用、代码块 —— 直接重写。**
