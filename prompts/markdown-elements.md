# Markdown 元素速查

> 写正文时用这页即可。目标是信息清楚，不要为了排版加载长示例库。

## 可用元素

| 场景 | 优先用法 |
|---|---|
| 主章节 | `## 小标题` |
| 章节内小点 | `### 小标题` |
| 关键判断 | `> 这一句单独拎出来` |
| 对比、参数、清单 | GFM 表格 |
| 步骤、并列要点 | 有序/无序列表 |
| 产品名、命令、参数 | 行内代码 `` `xxx` `` |
| 配置、命令片段 | fenced code block |
| 大段切换 | `---` |
| 提醒/避坑 | `> [!tip] 标题` 后接引用内容 |

## 写稿要求

- 不规定 Markdown 元素数量；正常连续叙述可以只用段落和自然小标题。
- 只有存在真实对比维度时才用表格，只有并列信息确实更清楚时才用列表。
- 引用块只放真正值得单独强调的判断或原话，不把普通总结强行做成引用。
- 能用表格说清楚的 A/B 对比，可以不生成正文图。
- 不为了炫技使用代码块；只有命令、配置、伪代码真的有用时才写。
- 图片不要直接重复正文信息；正文图由视觉阶段按需生成。
- 排版服务内容，不要为了“看起来丰富”制造表格、列表或引用。
